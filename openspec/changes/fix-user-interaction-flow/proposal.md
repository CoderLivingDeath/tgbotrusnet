## Why

Текущая реализация имеет критический баг: при команде `/start` пользователю показывается сообщение "Требуется авторизация", хотя для обычных пользователей авторизация не нужна. Бот должен приветствовать пользователей и показывать меню навигации без авторизации.

## What Changes

- Исправить порядок регистрации обработчиков: user handlers должны обрабатывать `/start` до admin/operator middleware
- Реализовать полноценный приветственный диалог с выбором категорий
- Обеспечить корректную маршрутизацию: пользователи видят меню, админы - команды

## Capabilities

### New Capabilities
- `user-welcome-flow`: Приветствие и навигация для обычных пользователей

### Modified Capabilities
- `faq-system`: Интеграция с приветственным потоком
- `operator-chat`: Эскалация диалога к оператору
- `logging`: Логирование с категориями

## Impact

- Изменения в `src/index.ts` - порядок middleware
- Изменения в `src/handlers/user/faq.ts` - приветственное меню
- Изменения в `src/middleware/auth.ts` - только для admin/operator команд
