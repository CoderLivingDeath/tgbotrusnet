## Context

Текущая проблема: при отправке `/start` обычным пользователем бот требует авторизацию. Это происходит из-за неправильного порядка регистрации middleware - admin/operator обработчики перехватывают все команды включая `/start`.

Существующая архитектура:
- `adminComposer` и `operatorComposer` используют `createAdminAuthMiddleware()` и `createOperatorAuthMiddleware()`
- Эти middleware вызывают `ctx.reply()` и НЕ вызывают `next()`, блокируя дальнейшую обработку
- Но проблема в том, что команда `/start` должна быть доступна всем без авторизации

## Goals / Non-Goals

**Goals:**
- Исправить баг с авторизацией при `/start` для обычных пользователей
- Реализовать приветственное меню с выбором категорий
- Обеспечить правильную маршрутизацию команд

**Non-Goals:**
- Изменение архитектуры авторизации админов/операторов
- Добавление новых функций (только исправление)

## Decisions

### Решение 1: Использовать явные команды с префиксом
- Admin команды: `/admin login`, `/admin stats` и т.д.
- Operator команды: `/operator login`, `/operator chats` и т.д.
- User команды: `/start`, `/menu`, `/search`, `/support`, `/end`

### Решение 2: Разделить обработчики по командам
- Composer'ы для admin/operator активируются только для своих команд
- User обработчики работают для всех остальных команд

### Выбранное решение
Использовать Telegraf `command()` с указанием конкретной команды вместо глобального перехвата. Это уже реализовано в коде, но есть проблема с порядком middleware.

## Risks / Trade-offs

- **Риск**: Новая путаница с командами - Mitigaton: использовать четкие префиксы `/admin`, `/operator`, `/user`
- **Риск**: Пользователи могут случайно использовать admin команды - Mitigaton: команды требуют авторизации
